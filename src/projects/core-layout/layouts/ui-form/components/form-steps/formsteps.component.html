{{'Form steps' | translate}}

Directives

{{'Form steps allow user to create forms with some sort of form flow (without refreshing your page). For example, if you had a large form for entering user data, you could use a form wizard to divide it into a series of related steps. This has the advantage of not overwhelming users with a really long form and also giving them some indication of their progress as they enter their information.' | translate}}

{{'How to use' | translate}}

{{'There is several ways to use the Form Steps component' | translate}}:

  1. {{'Slideshow' | translate}}: {{'used to separate screen from each other.' | translate}}
  2. {{'Single form' | translate}}: {{'used to separate parts of a form.' | translate}}
  3. {{'Mix of forms & texts' | translate}}: {{'used to have a full step wizard compose of textual screens and differents forms.' | translate}}

{{'Configuration' | translate}}

{{'Formsteps use an Array of Objects to set himself.' | translate}}

  1. {{'Single Form' | translate}}: {{'Only one object.' | translate}}
  2. {{'Slideshow & Mix of forms & texts' | translate}}: {{'One object per screen.' | translate}}

{{'Slideshow configuration' | translate}}:

{{'A slideshow needs a collection of objects (one by screen) inside an array. The title is optionnal.' | translate}}

  1. {{'title'}}: string => {{'each screen can have a title. For each screen name those titles like this: title1, title2, etc.' | translate}} ({{'optional' | translate}})
  2. {{'startHere'}}: boolean => {{'used to start directly to a specific step.' | translate}} ({{'optional' | translate}})
  3. {{'redirect'}}: boolean => {{'used to redirect user to a specific url instead of showing "Finished" screen.' | translate}} ({{'optional' | translate}})
  4. {{'goNext'}}: number => {{'used to go directly to a specific step.' | translate}} ({{'optional' | translate}})
  5. {{'goPrevious'}}: number => {{'used to go back directly to a specific step.' | translate}} ({{'optional' | translate}})
  6. {{'forceNext'}}: boolean => {{'boolean to allow user to go to next screen without fill the fields.' | translate}} Not used in "Slideshows"
  7. {{'submitted'}}: boolean => {{'used to avoid re-saving a form.' | translate}} Not used in "Slideshows"
  8. {{'form'}}: object => {{'used to validate the form and each formGroup.' | translate}} Not used in "Slideshows"
  9. {{'id'}}: string => {{'id of the form tag to link buttons to forms.' | translate}} Not used in "Slideshows"

{{'[formsteps]="[{title: \'Title 1\'}, {startHere: true}, {title: \'Title 3\'}]"'}}

{{'Single form configuration' | translate}}:

{{'Single form is based on <section> to separate each screen and FormGroup to valid each screen.' | translate}}

  1. {{'form'}}: object => {{'used to validate the form and each formGroup.' | translate}}
  2. {{'id'}}: string => {{'id of the form tag to link buttons to forms.' | translate}} Not used in "Single Forms"
  3. {{'forceNext'}}: boolean => {{'boolean to allow user to go to next screen without validators verification.' | translate}} ({{'optional' | translate}})
  4. {{'titleN'}}: string => {{'each screen can have a title. For each screen name those titles like this: title1, title2, etc.' | translate}} ({{'optional' | translate}})
  5. {{'redirect'}}: boolean => {{'used to redirect user to a specific url instead of showing "Finished" screen.' | translate}} ({{'optional' | translate}})
  6. {{'goNext'}}: number => {{'used to go directly to a specific step.' | translate}} ({{'optional' | translate}}) Not used in "Single Forms"
  7. {{'goPrevious'}}: number => {{'used to go back directly to a specific step.' | translate}} ({{'optional' | translate}}) Not used in "Single Forms"
  8. {{'startHere'}}: boolean => {{'used to start directly to a specific step.' | translate}} ({{'optional' | translate}}) Not used in "Single Forms"
  9. {{'submitted'}}: boolean => {{'used to avoid re-saving a form.' | translate}} Not used in "Single Forms"

{{'[formsteps]="[{form: myForm, forceNext: true, title1: \'Intro\', title2: \'Form\', title3: \'Submit\', redirect: \'ui-layouts/tooltips\'}]"'}}

{{'Mix of forms & texts' | translate}}:

{{'For a mix of forms and HTML screens, you have almost the same configuration than Slideshows (one object per screen), but this time we have to pass two new parameters: FORM & ID.' | translate}}

  1. {{'form'}}: object => {{'used to validate the form and each formGroup.' | translate}}
  2. {{'id'}}: string => {{'id of the form tag to link buttons to forms.' | translate}}
  3. {{'title'}}: string => {{'each screen can have a title.' | translate}} ({{'optional' | translate}})
  4. {{'startHere'}}: boolean => {{'used to start directly to a specific step.' | translate}} ({{'optional' | translate}})
  5. {{'redirect'}}: boolean => {{'used to redirect user to a specific url instead of showing "Finished" screen.' | translate}} ({{'optional' | translate}})
  6. {{'goNext'}}: number => {{'used to go directly to a specific step.' | translate}} ({{'optional' | translate}})
  7. {{'goPrevious'}}: number => {{'used to go back directly to a specific step.' | translate}} ({{'optional' | translate}})
  8. {{'submitted'}}: boolean => {{'used to avoid re-saving a form.' | translate}} ({{'optional' | translate}})
  9. {{'forceNext'}}: boolean => {{'boolean to allow user to go to next screen without fill the fields.' | translate}} ({{'optional' | translate}}) Not used in "Mix of forms & texts"
